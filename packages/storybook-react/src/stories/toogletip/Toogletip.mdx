# Toogletip

Los Toogletip poseen la misma lógica de aparición que posee el Tooltip con la diferencia que este componente muestra el contenido en
una card con la posibilidad de agregar enlaces y botones de acción.
Muestran información que no es imprescindible para la experiencia. Por eso se oculta detrás de una acción.

<br />

<Primary />
## Visibilidad

### $visible

El componente posee una prop llamada $visible que recibe un valor de tipo boolean, si es true, se muestra el Toogletip, si es false (por default) el Toogletip se mantendrá oculto

### $onToogletipClose

El componente posee una prop llamada $onToogletipClose que sirve para cerrar y ocultar la visibilidad del componente

<br />
<br />

## Información Importante

El tooltip intentará en lo posible no superar los limites del viewport. Por lo cual si se le ordena renderizarse
a la izquierda de un elemento y no tiene espacio, se ubicara a la derecha. Es por eso que a veces su comportamiento
puede parecer que no obedece las props indicadas.

### Información importante sobre el orden de renderizado de botones de acción

El componente aplica una restricción en la cual limita a solo 2 componentes de acción.
Para eso prioriza los componentes de la siguiente manera: Link - Link - Button - Button

Es decir:

- Si en las props se le pasan 2 o mas Links y [n] cantidad de Buttons, el Toogletip mostrara solo dos primeros links.
- Si solo muestra un Link y 1 o + cantidad de Buttons, el toogletip mostrará el link y el primer Button.
- Si no hay links y hay 1 o + cantidad de buttons, el toogletip mostrara los 2 primeros Buttons.

### Toogletip con steps

El componente posee una prop llamada $steps que recibe un array de objectos con propiedades 'title' y 'label'.
Si esta prop se define, el Toogletip mostrara un label indicando el paso actual y botones de Anterior y Siguiente para recorrer
los diferentes steps o estadíos del mismo.
